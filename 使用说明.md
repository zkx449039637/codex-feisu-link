# codex-feishu-link 使用说明

本文档面向日常使用者，而不是开发者。目标是让你在 Windows 上稳定使用 `codex-feishu-link`，并在需要时快速迁移到另一台电脑。

## 1. 这是什么

`codex-feishu-link` 会把飞书机器人消息转成对本机 Codex CLI 的控制命令。

你可以把它理解为：

- 飞书里发指令
- 本机收到指令
- 本机在指定项目目录里执行 Codex 任务
- 再把结果状态回到飞书

## 2. 当前机器的必备条件

这台机器要能正常运行，至少需要：

- 一个可用的 `config.local.json`
- 飞书机器人的 `app_id`
- 飞书机器人的 `app_secret`
- 允许控制机器人的 `allowed_user_ids`
- 至少一个项目目录 `workdir`
- 本机可用的 Codex CLI
- 本仓库的 `.venv`

本仓库当前默认优先使用：

- 配置文件：`config.local.json`
- Python：`.venv\Scripts\python.exe`
- Codex 启动方式：`node node_modules\@openai\codex\bin\codex.js`

## 3. 最常用的启动和停止方式

### 3.1 推荐日常使用：后台启动

后台启动后，不需要一直保留当前终端窗口。

PowerShell 启动：

```powershell
cd "D:\Codex Projects\codex-feisu-link"
.\start-background.ps1
```

也可以直接双击：

- `start-background.cmd`

### 3.2 查看后台状态

```powershell
cd "D:\Codex Projects\codex-feisu-link"
.\service-status.ps1
```

### 3.3 停止后台服务

只停止后台 PID 文件记录的那个实例：

```powershell
cd "D:\Codex Projects\codex-feisu-link"
.\stop-background.ps1
```

### 3.4 一键停止本机全部实例

如果你曾经手动前台启动过，或者后台启动过多次，推荐直接用这个：

```powershell
cd "D:\Codex Projects\codex-feisu-link"
.\stop-all.ps1
```

这个脚本会停止本机所有 `python -m codex_feishu_link ...` 实例。

### 3.5 前台调试启动

用于排障。窗口不要关，关了服务就停。

```powershell

.\run-service.ps1 -Mode service
```

如果要停止前台服务，直接在该窗口按：

```text
Ctrl+C
```

## 4. 飞书里怎么用

启动服务后，直接在飞书给机器人发送纯文本命令即可。

支持的命令如下。

### 4.1 `projects`

用途：查看当前这台机器可控制的项目列表。

示例：

```text
projects
```

适合在以下场景使用：

- 刚启动服务，确认机器人在线
- 查看当前配置里有哪些项目

### 4.2 `tasks [project]`

用途：查看任务列表。

示例：

```text
tasks
tasks codex-feishu-link
```

说明：

- 不带项目名时，显示全部项目的任务
- 带项目名时，只显示该项目的任务

### 4.3 `new <project> <description...>`

用途：在指定项目中创建一个新任务。

示例：

```text
new codex-feishu-link 修复后台启动脚本
new codex-feishu-link "检查服务停止逻辑并补充文档"
```

说明：

- `project` 必须是 `projects` 里能看到的项目名
- 后面的描述会作为任务提示词的一部分进入本机执行

### 4.4 `status <task_id>`

用途：查看单个任务状态。

示例：

```text
status task_123456abcdef
```

### 4.5 `logs <task_id>`

用途：查看任务最近日志。

示例：

```text
logs task_123456abcdef
```

适合排查：

- 卡住
- 没出结果
- 想看最近一步在做什么

### 4.6 `diff <task_id>`

用途：查看任务变更摘要或 diff 信息。

示例：

```text
diff task_123456abcdef
```

### 4.7 `continue <task_id> [instruction...]`

用途：继续一个已存在任务，并可追加新的说明。

示例：

```text
continue task_123456abcdef
continue task_123456abcdef 请补充中文使用文档
```

说明：

- 如果任务是 `queued`、`paused` 或 `waiting_confirmation`，会尝试恢复运行
- 如果你追加了说明，会作为 follow-up instruction 记录到任务里

### 4.8 `pause <task_id>`

用途：暂停任务。

示例：

```text
pause task_123456abcdef
```

### 4.9 `resume <task_id>`

用途：恢复已暂停任务。

示例：

```text
resume task_123456abcdef
```

### 4.10 `stop <task_id>`

用途：停止任务。

示例：

```text
stop task_123456abcdef
```

### 4.11 `confirm <task_id> yes|no`

用途：对等待确认的任务给出确认结果。

示例：

```text
confirm task_123456abcdef yes
confirm task_123456abcdef no
```

说明：

- `yes`：同意，任务继续
- `no`：拒绝，任务转为暂停

### 4.12 `snapshot <task_id>`

用途：查看任务完整快照。

示例：

```text
snapshot task_123456abcdef
```

快照通常会包含：

- 任务 ID
- 项目名
- 当前状态
- 原始提示词
- 工作目录
- 分支
- 摘要
- 最近日志
- diff
- 是否正在等待确认

## 5. 任务状态说明

系统里常见的任务状态如下：

- `queued`：已排队，尚未开始
- `running`：正在运行
- `waiting_confirmation`：等待人工确认
- `paused`：已暂停
- `completed`：已完成
- `failed`：执行失败
- `stopped`：已停止

一般建议：

- 想看机器现在在做什么：`status` 或 `logs`
- 想继续一个停下来的任务：`continue` 或 `resume`
- 想看这次改了什么：`diff`

## 6. 配置文件怎么理解

日常只需要关注：

- `feishu.app_id`
- `feishu.app_secret`
- `feishu.allowed_user_ids`
- `projects`

配置文件位置：

- `config.local.json`

建议原则：

- 不要删掉整个文件结构
- 只改你需要改的值
- 优先保留 `config.local.json`
- 不要拿 `config.example.json` 当真实配置直接跑

### 6.1 `allowed_user_ids` 是什么

它不是机器人的 ID，而是“允许操作这个机器人”的飞书用户 ID 白名单。

示例：

```json
"allowed_user_ids": [
  "ou_xxx",
  "ou_yyy"
]
```

含义：

- 只有这些飞书用户发给机器人消息时，机器人才会执行命令

## 7. 常见操作建议

### 7.1 每天使用

推荐流程：

1. 双击 `start-background.cmd`
2. 飞书里发送 `projects`
3. 开始正常发任务命令
4. 不用时运行 `.\stop-background.ps1`

### 7.2 怀疑有残留进程

直接运行：

```powershell
.\stop-all.ps1
```

### 7.3 飞书没反应

按顺序检查：

1. `.\service-status.ps1`
2. 查看 `.runtime\service\service.stdout.log`
3. 查看 `.runtime\service\service.stderr.log`
4. 飞书里再发一次 `projects`
5. 如果仍异常，用前台模式启动 `.\run-service.ps1 -Mode service` 看报错

### 7.4 服务已经停了但飞书还在回复

最常见原因：

- 本机还有别的残留实例
- 另一台电脑也在运行同一个飞书机器人

先执行：

```powershell
.\stop-all.ps1
```

如果本机停干净后飞书还在回复，说明大概率是另一台机器仍在线。

## 9. 换电脑时的快速配置方法

把这个仓库当作“多电脑快速部署模板”使用即可。

### 9.1 哪些内容可以复用

这些内容通常直接复制过去就能继续用：

- 仓库代码本身
- 飞书命令协议
- 启动脚本
- 后台脚本
- 文档

### 9.2 哪些内容必须按新机器调整

这些通常是换电脑时最需要重新确认的：

- `config.local.json`
- 飞书 `app_id`
- 飞书 `app_secret`
- `allowed_user_ids`
- 每个项目的 `workdir`
- 本机 `codex_executable` 和 `codex_arguments`

### 9.3 换电脑最快步骤

1. 把仓库复制到新机器
2. 打开 PowerShell，进入仓库根目录
3. 创建 `.venv`
4. 安装项目和依赖
5. 生成新的 `config.local.json`
6. 填机器专属值
7. 启动服务
8. 飞书里发送 `projects` 验证

示例命令：

```powershell
cd "D:\Codex Projects\codex-feisu-link"
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -e . lark-oapi
.\init-machine.ps1 -Force -MachineName $env:COMPUTERNAME -ProjectName "codex-feishu-link" -ProjectPath "D:\Codex Projects\codex-feisu-link"
```

然后编辑 `config.local.json`，至少填：

- `feishu.app_id`
- `feishu.app_secret`
- `feishu.allowed_user_ids`
- `projects.<project>.workdir`

最后后台启动：

```powershell
.\start-background.ps1
```

再去飞书发：

```text
projects
```

### 9.4 复用同一个飞书机器人时的注意事项

如果两台机器使用同一个飞书机器人：

- 同一时刻最好只保留一台在线
- 切换到新机器前，先在旧机器执行 `.\stop-all.ps1`

如果你想多台机器同时在线：

- 建议一机一个飞书机器人
